# Security Policies and Procedures

This document outlines security procedures and general policies for the
`ai-defense-langchain-middleware` project.

## Disclosing a Security Issue

The maintainers take all security issues seriously. Thank you for improving
the security of this project. We appreciate responsible disclosure and will
make every effort to acknowledge your contributions.

This project leverages GitHub's private vulnerability reporting.

To learn more about this feature and how to submit a vulnerability report,
review [GitHub's documentation on private reporting](https://docs.github.com/code-security/security-advisories/guidance-on-reporting-and-writing-information-about-vulnerabilities/privately-reporting-a-security-vulnerability).

When filing a report, please include:

- A detailed description of the issue
- Steps required to reproduce the issue
- Versions of the project that may be affected
- If known, any mitigations for the issue

A maintainer will acknowledge the report within three (3) business days and
will send a more detailed response within an additional three (3) business
days indicating next steps.

If you have been unable to submit a report via GitHub or have not received a
response during the allotted window, please reach out via the
[Cisco Open security contact email](mailto:oss-security@cisco.com).

## Vulnerability Management

When the maintainers receive a disclosure report, they will assign it to a
primary handler. This person will coordinate the fix and release process:

- Confirming the issue
- Determining affected versions
- Auditing code to find similar problems
- Preparing fixes for all releases under maintenance

## Suggesting Changes

If you have suggestions on how this process could be improved, please submit
an issue or pull request.
